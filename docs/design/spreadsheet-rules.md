# 스프레드시트 운영 규칙

> 앱(CloudPocket)과 Google Sheets 연동 시 지켜야 할 규칙

## 앱에서만 해야 하는 것

| 작업 | 이유 |
|------|------|
| 카테고리/소분류 추가·수정·삭제 | 시트에서 직접 변경하면 앱 매칭 실패 |
| 결제수단 추가·수정·삭제 | 동일 |
| 거래 입력·수정·삭제 | 앱이 데이터 소스, 시트는 백업 |

## 시트에서 해도 되는 것

| 작업 | 비고 |
|------|------|
| 데이터 조회/확인 | 읽기만 |
| 차트/서식 변경 | 앱 동기화에 영향 없음 |
| 셀에 메모 추가 | 동기화 범위 밖 |
| 수식 확인 | 수식 셀에 값 쓰지 않기 |

## 절대 하지 말 것

| 작업 | 위험 |
|------|------|
| 행/열 삽입·삭제 | 앱의 셀 범위 매핑이 깨짐 |
| 카테고리명 직접 변경 | 앱에 저장된 기존 거래와 매칭 실패 |
| 수식 셀에 값 덮어쓰기 | 월간 요약, 연간 대시보드 수식 파괴 |
| 거래 영역 중간에 빈 행 삽입 | 앱이 읽기 중단 |

## 잠재적 위험 & 대응

| 위험 | 대응 |
|------|------|
| 시트에서 카테고리명 변경 → 매칭 실패 | 앱에서 먼저 변경 후 내보내기 |
| 여러 기기 동시 편집 → 덮어쓰기 | 내보내기 전 항상 가져오기 먼저 |
| 연도 전환 시 시트 미생성 → export 실패 | MS6에서 자동 시트 생성 구현 예정 |
| 빈 행 중간 삽입 → 읽기 중단 | 행 삭제 대신 내용만 지우기 |

## 시트 분리 규칙 (2026-03-18)

| 시트 | 역할 | 비고 |
|------|------|------|
| `설정-카테고리` | 지출 대분류+소분류 매트릭스 (A1:P20) | `(필수)설정 시트`에서 분리 |
| `설정-결제수단` | 신용카드(A열), 체크/현금(B열) (A1:A10, B1:B10) | `(필수)설정 시트`에서 분리 |
| `(필수)설정 시트` | 저축상품, 은행계좌, 수입분류 (미구현 기능) | 유지 — 향후 추가 분리 |

- 각 설정 시트는 **A1 기반 범위**를 사용하여 셀 범위 충돌 방지
- `importAll()`은 연도 셀을 읽지 않음 — `year` 파라미터 또는 현재 연도 사용
- 새 설정 시트 추가 시 동일 패턴 적용: `설정-{도메인}` + A1 기반 범위

## 향후 개선 권장

| # | 항목 | 이유 |
|---|------|------|
| 1 | 수입에 결제수단 컬럼 추가 | 계좌 추적 |
| 2 | 거래ID 숨김 컬럼 (M열 뒤) | 앱↔시트 동일 거래 식별 |
| 3 | 연간 대시보드에 예산 행 | 예산 vs 실적 비교 |
| 4 | 저축상품/은행계좌/수입분류 시트 분리 | 해당 기능 구현 시 |
