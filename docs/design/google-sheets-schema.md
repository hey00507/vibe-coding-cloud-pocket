**헤더 (Row 5)**:

| F열 | G열 | H열 | I열 | J열 | K열 | L열 |

### 3.6 영역 E: 카테고리별 지출 요약 (Row 5+, Col N~T)

두 개 열 그룹으로 구성:

**그룹 1 (Col N~P)**: 대분류 | 소분류 | 지출금액
**그룹 2 (Col R~T)**: 대분류 | 소분류 | 지출금액

각 대분류별 소분류 지출 합계와 대분류 합계를 정리한 피벗 테이블 형태.

### 3.7 영역 F: 월간 캘린더 (Row 6+, Col V~AB)

| V열 | W열 | X열 | Y열 | Z열 | AA열 | AB열 |
|-----|-----|-----|-----|-----|------|------|
| 일 | 월 | 화 | 수 | 목 | 금 | 토 |

- 각 주별로 2행 사용: 첫 행은 날짜, 둘째 행은 해당 일자의 총 지출 금액
- 최대 6주 표시

## 5. 현재 앱 모델과의 갭 분석

### 5.1 카테고리 구조

| 항목 | 현재 앱 | Google Sheets 템플릿 |
|------|---------|---------------------|
| 계층 구조 | **1단계** (flat) | **2단계** (대분류 → 소분류) |
| 카테고리 타입 | `income` / `expense` | **지출만 계층화**, 수입은 별도 분류 |
| 이모지 | `icon` 필드 (선택) | 소분류명에 이모지 포함 (예: ☕간식/카페) |
| 색상 | `color` 필드 (선택) | 없음 (조건부 서식으로 처리) |

### 5.2 거래(Transaction) 구조

| 필드 | 현재 앱 | Google Sheets 템플릿 |
|------|---------|---------------------|
| id | `string` (UUID) | 없음 (행 순서로 관리) |
| type | `income` / `expense` | 시트 분리 (수입=별도 영역, 지출=거래 입력 영역) |
| amount | `number` | `number` |
| date | `Date` 객체 | `number` (해당 월의 일자만, 예: `2`) |
| categoryId | `string` (단일 카테고리) | **대분류 + 소분류** (2개 필드) |
| paymentMethodId | `string` | `string` (카드명 직접 기록) |
| memo | `string` (선택) | `세부사항` (선택) |
| 남은 잔액 | 없음 | 자동계산 필드 |

### 5.3 신규 도메인 (앱에 없는 개념)

| 도메인 | 설명 | 우선순위 |
|--------|------|---------|
| **수입 관리** | 수입 분류별 목표금액/실적 관리 | 높음 |
| **저축 관리** | 저축 항목별 목표/실적, 보험료 관리 | 중간 |
| **저축 상품** | 상품명, 금리, 은행, 가입/만기일 등 상세 관리 | 낮음 |
| **은행/계좌 관리** | 은행별 계좌, 용도, 자산 현황 | 낮음 |
| **연간 요약** | 월별 수입/지출/저축 매트릭스 | 자동계산 |
| **월간 캘린더** | 일별 지출 합계 캘린더 뷰 | 이미 구현됨 |

## 6. 데이터 모델 확장 제안

### 6.1 카테고리 모델 변경

```typescript
// 현재
interface Category {
  id: string;
  name: string;
  type: TransactionType;
  icon?: string;
  color?: string;
}

// 확장안: 대분류/소분류 계층 구조
interface Category {
  id: string;
  name: string;           // 대분류명 (예: "식비")
  type: TransactionType;
  icon?: string;
  color?: string;
}

interface SubCategory {
  id: string;
  categoryId: string;     // 부모 대분류 ID
  name: string;           // 소분류명 (예: "간식/카페")
  icon?: string;          // 이모지 (예: "☕")
}
```

### 6.2 거래 모델 변경

```typescript
// 현재
interface Transaction {
  id: string;
  type: TransactionType;
  amount: number;
  date: Date;
  categoryId: string;
  paymentMethodId: string;
  memo?: string;
}

// 확장안: 소분류 추가
interface Transaction {
  id: string;
  type: TransactionType;
  amount: number;
  date: Date;
  categoryId: string;        // 대분류 ID
  subCategoryId?: string;    // 소분류 ID (선택)
  paymentMethodId: string;
  memo?: string;
}
```

### 6.3 수입 모델 (신규)

```typescript
interface IncomeCategory {
  id: string;
  name: string;              // 월급, 상여급, 금융소득, 부수입, 기타
  targetAmount: number;      // 월 목표금액
}

// 수입 거래는 기존 Transaction에서 type='income'으로 처리 가능
```

### 6.4 저축 모델 (신규)

```typescript
interface SavingsItem {
  id: string;
  name: string;              // 저축 분류명 (주택청약, 은행명 등)
  targetAmount: number;      // 월 목표금액
}

interface SavingsProduct {
  id: string;
  name: string;              // 상품명
  status: 'active' | 'pending';  // 진행중 / 시작전
  interestRate: number;      // 연 적용 금리
  bank: string;              // 은행명
  startDate?: Date;          // 가입일
  endDate?: Date;            // 만기일
  monthlyAmount: number;     // 월 납입액
  paidMonths: number;        // 납입 개월 수
  currentAmount: number;     // 현재 금액
  memo?: string;             // 비고
}
```

### 6.5 결제수단 모델 변경

## 7. Google Sheets 동기화 매핑

앱 ↔ Google Sheets 간 데이터 변환 시 사용할 매핑 규칙.

### 7.1 지출 거래 매핑 (앱 → 시트)

| 앱 필드 | 시트 열 | 변환 규칙 |
|---------|--------|-----------|
| `date` | F열 (날짜) | `Date` → 일자 숫자만 추출 (예: `2`) |
| `categoryId` → `category.name` | G열 (대분류) | ID → 이름 변환 |
| `subCategoryId` → `subCategory.icon + subCategory.name` | H열 (소분류) | ID → 이모지+이름 변환 |
| `amount` | I열 (금액) | 그대로 |
| `paymentMethodId` → `paymentMethod.name` | J열 (결제수단) | ID → 이름 변환 |
| `memo` | K열 (세부사항) | 그대로 |
| *(자동계산)* | L열 (남은 잔액) | 시트 수식으로 계산 |

### 7.2 시트 → 앱 가져오기 시

| 시트 열 | 앱 필드 | 변환 규칙 |
|--------|---------|-----------|
| F열 (날짜) | `date` | 시트명(월) + 일자 → `Date` 객체 생성 |
| G열 (대분류) | `categoryId` | 이름 → ID 매칭 (없으면 자동 생성) |
| H열 (소분류) | `subCategoryId` | 이모지 제거 후 이름 → ID 매칭 |
| I열 (금액) | `amount` | 그대로 |
| J열 (결제수단) | `paymentMethodId` | 이름 → ID 매칭 (없으면 자동 생성) |
| K열 (세부사항) | `memo` | 그대로 |

### 7.3 시트별 읽기/쓰기 범위

| 시트 | 읽기 범위 | 쓰기 범위 | 용도 |
|------|----------|----------|------|
| `설정-카테고리` | A1:P20 | A1:P20 | 카테고리 대분류/소분류 매트릭스 (20행×16열) |
| `설정-결제수단` | A1:A10, B1:B10 | A1:A10, B1:B10 | 결제수단 (A=신용, B=체크/현금) |
| `(필수)설정 시트` | B25:B29 | - | 수입 분류 가져오기 (미구현) |
| `(필수)설정 시트` | D4:N16 | D4:N16 | 저축 상품 관리 (미구현) |
| `{월}` | F5:K1000 | F6:K1000 | 지출 거래 읽기/쓰기 |
| `{월}` | A13:E20 | B15:D19 | 수입 내역 읽기/쓰기 |
| `{월}` | A21:E34 | B22:D33 | 저축 내역 읽기/쓰기 |
| `2026` | A30:O44 | - | 연간 요약 (읽기 전용, 수식 자동계산) |
