## AsyncStorage 연동 (데이터 영속성)

### 구현 내용

#### StorageService 추상화 레이어
- `IStorageService` 인터페이스 정의 (save, load, saveValue, loadValue, remove, clearAll)
- `StorageService` 구현체: AsyncStorage 래퍼, Date 필드 자동 복원 (reviver)
- `storageKeys.ts`: 8개 도메인별 저장소 키 상수 정의

#### 7개 데이터 서비스 영속성 추가
- **Write-Through Cache 패턴**: Map 변경 후 AsyncStorage에 fire-and-forget 저장
- 각 서비스에 `hydrate()` / `persist()` 메서드 추가
- `hydrate()`: AsyncStorage → Map 복원 + idCounter 복원
- `persist()`: Map → AsyncStorage 저장 (storageService가 null이면 no-op)
- Date 필드 처리: TransactionService(`date`), SavingsService(`startDate`, `endDate`)

#### SeedService 영속성
- `seeded` 플래그 AsyncStorage 저장/복원
- 앱 재시작 시 이미 seeded면 재시딩 방지

#### AppInitializer + App.tsx
- `initializeApp()`: SeedService hydrate → 7개 서비스 병렬 hydrate → 미시딩 시 seedAll
- App.tsx: 로딩 상태 (ActivityIndicator) → 초기화 완료 후 메인 화면 렌더링
