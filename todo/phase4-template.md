# Phase 4: 템플릿 완전 활용

## 개요
Excel 가계부 템플릿의 모든 시트/영역을 앱에서 시각화하고 편의 기능으로 활용한다.

> **선행 조건**: Phase 3 Google Sheets 연동 완료 (GCP 설정 + 실기기 검증)

---

## 핵심 목표

기존 서비스(SavingsService, BankAccountService, IncomeTargetService)는 구현+테스트 완료 상태이지만 **UI가 없다.**
템플릿의 각 영역에 대응하는 화면을 만들어 서비스를 활용한다.

---

## 1. 수입 관리 화면 (템플릿 Area B 대응)

**현재**: IncomeTargetService 존재, UI 없음
**목표**: 수입 카테고리별 목표 설정 + 실적 대비 시각화

### 1.1 수입 대시보드
- [ ] 월간 수입 목표 vs 실적 비교 카드
- [ ] 카테고리별 수입 목표 달성률 (프로그레스 바)
- [ ] 급여/상여/금융소득/부수입/기타 항목별 표시

### 1.2 수입 목표 설정 UI
- [ ] Settings에서 연간/월간 수입 목표 CRUD
- [ ] IncomeTargetService 연결

### 1.3 테스트
- [ ] 수입 대시보드 컴포넌트 테스트
- [ ] 목표 설정 UI 테스트

---

## 2. 저축 관리 화면 (템플릿 Area C 대응)

**현재**: SavingsService 존재, UI 없음
**목표**: 저축상품 현황 + 월별 납입 추적

### 2.1 저축 대시보드
- [ ] 저축상품 목록 (상품명, 은행, 이자율, 월납입액, 납입현황)
- [ ] 총 저축액 요약 카드
- [ ] 상태별 필터 (active/pending)

### 2.2 저축상품 CRUD UI
- [ ] Settings에서 저축상품 추가/수정/삭제
- [ ] SavingsService 연결

### 2.3 테스트
- [ ] 저축 대시보드 컴포넌트 테스트
- [ ] 저축상품 CRUD UI 테스트

---

## 3. 은행계좌/자산 관리 화면 (템플릿 설정 시트 대응)

**현재**: BankAccountService 존재, UI 없음
**목표**: 계좌별 잔액 + 총 자산 현황

### 3.1 자산 대시보드
- [ ] 계좌 목록 (은행, 계좌명, 잔액)
- [ ] 등급별(primary/secondary/savings_bank) 그룹핑
- [ ] 총 자산 요약 카드

### 3.2 계좌 CRUD UI
- [ ] Settings에서 계좌 추가/수정/삭제
- [ ] BankAccountService 연결

### 3.3 테스트
- [ ] 자산 대시보드 컴포넌트 테스트
- [ ] 계좌 CRUD UI 테스트

---

## 4. 시각화 강화 (템플릿 연간 대시보드 대응)

**현재**: AnnualDashboard 컴포넌트 존재 (기본 매트릭스)
**목표**: 템플릿의 연간 대시보드 시트를 완벽히 재현

### 4.1 연간 대시보드 강화
- [ ] 수입/저축/지출/잔여현금 행 추가
- [ ] 카테고리별 월간 지출 히트맵
- [ ] 연간 합계 컬럼

### 4.2 월간 요약 강화 (Area A 대응)
- [ ] 수입 총액, 지출 총액, 저축 총액, 잔여 현금을 한 눈에
- [ ] 전월 대비 증감 표시

### 4.3 카테고리별 집계 (Area E 대응)
- [ ] 대분류별 월간 지출 피벗 테이블
- [ ] 소분류 펼치기/접기

---

## 5. Settings 통합

현재 Settings에서 관리 가능한 것: 테마, 카테고리, 소카테고리, 결제수단
추가해야 할 것:

- [ ] 수입 목표 관리 섹션
- [ ] 저축상품 관리 섹션
- [ ] 은행계좌 관리 섹션
- [ ] Google Sheets 연동 (Phase 3에서 실동작 연결)

---

## 작업 순서 권장

```
1단계: 수입 관리 — IncomeTargetService UI 연결 (서비스 이미 있음)
2단계: 저축 관리 — SavingsService UI 연결 (서비스 이미 있음)
3단계: 자산 관리 — BankAccountService UI 연결 (서비스 이미 있음)
4단계: Settings 통합 — 위 3개를 Settings에 CRUD 추가
5단계: 시각화 강화 — 연간 대시보드, 월간 요약, 카테고리 피벗
```

> 1~3단계는 서비스가 이미 완성되어 있으므로 **UI만 만들면 됨** — 상대적으로 빠른 작업
